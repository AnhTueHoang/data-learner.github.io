Project Overview

The dataset contains information about telecom customers, including their demographics, and the services they subscribe to, contract type, billing information, and whether they have churned. I approached this as a supervised classification problem, where the target variable is Churn (Yes/No).

My main goals were to:

  • Understand what types of customers are more likely to churn.
  • Build a predictive model that flags at-risk customers.
  • Translate model results into simple, business-friendly insights.

Process & Approach

  • Loaded the Telco Customer Churn dataset into a Pandas DataFrame.
  • Cleaned the data by fixing the TotalCharges column and handling missing values.
  • Explored churn patterns using visualizations (e.g., churn distribution, tenure vs churn, contract type vs churn, monthly charges vs churn).
  • Separated the target variable Churn and encoded it as 0 (No) and 1 (Yes).
  • Split the dataset into training and test sets using an 80/20 split.
  • Built a preprocessing pipeline using ColumnTransformer to:
    • Standardize numeric features: tenure, MonthlyCharges, TotalCharges.
    • One-hot encode categorical features (contract type, internet service, payment method, etc.).
  • Combined preprocessing with a Logistic Regression model in a single scikit-learn Pipeline, ensuring consistent transformations for train and test data.
  • Trained the model and evaluated it using accuracy, precision, recall, F1 score, and a confusion matrix.

Key Analytical Findings

  • Class imbalance: About one-quarter of customers churned while the majority stayed. This makes churn a minority class, highlighting the challenge of accurately predicting it.
  • Tenure: Customers who churn typically have significantly shorter tenure. Many leave within their first 10 months, while customers who stay tend to have much longer relationships with the company.
  • Contract type: Month-to-month contracts show the highest churn rates, while one-year and two-year contracts are associated with much lower churn, indicating that contract commitment strongly reduces churn risk.
  • Monthly charges: Customers who churn tend to pay higher monthly charges. This suggests that expensive plans may feel less valuable or carry more financial risk, making customers more likely to leave.

Model Performance & Interpretation

I trained a Logistic Regression model using the preprocessed features and evaluated it on an 80/20 train-test split. The model reached an overall accuracy of 82%. Because churn prediction is imbalanced, accuracy alone is not enough â€” so I evaluated precision, recall, and F1 score for the churn class specifically.

  • Precision (churn = 1): 0.69 â€” When the model predicts a customer will churn, it is correct 69% of the time.
  • Recall (churn = 1): 0.60 â€” The model correctly identifies 60% of customers who actually churn.
  • F1 Score (churn = 1): 0.64 â€” This reflects a balanced performance between precision and recall.
  • Precision (non-churn = 0): 0.86 and Recall (non-churn = 0): 0.90 â€” The model is very strong at identifying customers who stay.

These results are typical for churn prediction problems, where the minority class (churners) is harder to identify. The confusion matrix shows that the model correctly predicted most non-churners (934 true negatives) and several churners (223 true positives), but it still missed some churners (150 false negatives).

This indicates that although the model is a solid starting point, future improvements could focus on boosting recall for the churn class through class weighting, oversampling techniques, or trying more advanced algorithms like Random Forest or XGBoost.
